1. js不能准确地表示三分之一的数值字面量，如0.3-0.2 != 0.2-0.1，因为0.3-0.2 = 0.0999999999998
2. 系统级、出乎意料的或者类似错误的值的空缺使用 undefined，而程序级、正常的或意料之中的值的空缺使用 null。
3. 删除属性存在的坑： 
    a = {n: {x: 2}}, 
    b = a.n; 
    delete a.n; 
这段代码执行之后，b.x 依然等于 2，原因是 {x:2} 这个对象被 a 和 b 同时引用，delete 指令只删除了 a 对它的引用，b 上的引用依然存在。这种问题有可能造成内存泄漏。
4. this指向，严格模式下，指向window的会指向undefined
5. JSON.stringify 接受三个参数，第三个参数可以设置空白字符来美化输出，第二个参数为 {Array|Function} 类型，如果为 Array 则用于过滤 key，如果为 Function 则可以对 value 做处理
